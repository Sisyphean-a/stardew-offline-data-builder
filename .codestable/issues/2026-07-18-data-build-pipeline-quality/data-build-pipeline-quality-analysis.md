# 数据构建质量根因分析

## 根因

1. 旧格式解析一律按 `/` 拆分，且缺少成就、鞋类和怪物的真实显示字段映射；成就的 `^` 格式因此退化为数字 ID。
2. 图片元数据只处理通用纹理；大型可制作物、家具、鞋类没有默认图集和真实格子尺寸，成就也没有官方 UI 矩形。缺少 `imageSource` 时图片管线静默跳过。
3. 翻译状态只判断中文值是否为真值，纯数字 ID 被错误计为 `complete`。
4. 构建摘要把 `success` 写死为 `true`，并在有解析、资源或覆盖错误时照常生成包。
5. 独立 `package` 从数据库临时重建不完整摘要，丢失扩展类型、质量、游戏版本、源哈希和生成时间。
6. 现有夹具主要覆盖通用 `entries` 结构，未同时覆盖真实旧格式、实际图集裁切和失败质量门禁。

## 修复原则

- 以本机官方数据和游戏程序集公开行为为字段、图集和尺寸依据，不用人工映射伪造成功。
- 将可展示性、必需图片、类型展示名和数据错误统一纳入持久化质量契约；质量失败保留诊断产物但禁止发布包。
- 用真实格式正向回归和必需图片缺失的反向回归约束行为。

## 结论

六项发现共享同一个发布完整性缺口，必须一起修复；仅修解析或仅修报告都会留下“不可用产物被标成功”的路径。
