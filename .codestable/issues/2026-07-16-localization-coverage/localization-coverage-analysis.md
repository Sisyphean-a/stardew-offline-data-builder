# 真实数据本地化覆盖不足根因分析

## 1. 问题定位

| 关键位置 | 说明 |
|---|---|
| `src/builder/parsers/official.py:73` | 映射记录不把大写 `Name` 作为显示名，特别订单的 `[Caroline_Name]` 未进入标准字段。 |
| `src/builder/parsers/official.py:95-101` | 旧式斜杠记录统一使用回退 ID；家具、任务和收集包的显示字段未投影，料理/制作配方也未保留显式显示名与产物关联的优先级。 |
| `src/builder/parsers/localization.py:181-220` | 仅解析 `[LocalizedText ...]`，且资产路径未折叠重复分隔符，无法解析怪物任务中的 `Strings\\Locations`。 |
| `src/builder/parsers/localization.py:33-44` | 没有特别订单占位符对应的 `Strings/SpecialOrderStrings` 解析路径。 |
| `src/builder/parsers/official_assets.py:11` | `TailoringRecipes` 被误归为 `crafting_recipe`，混淆可展示配方和技术组合规则。 |
| `src/builder/pipeline/normalize.py:50` | 只要没有中文名称就标记 `missing`，未区分日程、裁缝规则、商店 ID、掉落规则等不要求翻译的记录。 |

## 2. 失败路径还原

**正常路径**：官方数据记录 → 解析出显示文本或可关联产物 → 从对应 `Strings/*.zh-CN.json` 解析中文 → 标准化实体写入中文字段 → 报告仅统计确实需要中文但缺失的实体。

**失败路径**：家具、任务、收集包、配方和特别订单进入通用解析 → 名称被回退成 ID、产物名或保留占位符 → 本地化解析无法命中正确字符串表 → 标准化器将技术记录和用户可见记录一并标记为 `missing`。

**分叉点**：`official.py:95-101` 未按旧式资产结构投影名称/产物；`localization.py:219-220` 未规范双反斜杠资产路径；`normalize.py:50` 将所有无中文名记录等同于缺汉化。

## 3. 根因

**根因类型**：数据格式。

**根因描述**：真实官方资产同时存在映射、旧式斜杠记录和字符串占位符三种本地化形式。当前实现只完整覆盖了标准 `[LocalizedText asset:key]` 的单反斜杠形式，未把家具、任务、收集包和配方的旧式字段、特别订单的短占位符及双反斜杠路径纳入同一解析路径；配方关联又无条件以产物名覆盖显式名；随后过于粗粒度的翻译状态规则放大了告警数量。

**是否有多个根因**：是。主因是官方格式适配不完整；次因是翻译状态未建模“不适用”。

## 4. 影响面

- **影响范围**：真实官方构建的家具、特别订单、怪物任务、旧式任务、旧式收集包、普通制作配方、部分料理与报告统计。
- **潜在受害模块**：SQLite 搜索、`.svdata` 消费端中文展示、构建报告和扩展类型覆盖统计。
- **数据完整性风险**：无原始数据丢失；错误仅发生在标准化展示字段和告警状态，原始属性仍保存在 `extra_json`。
- **严重程度复核**：维持 **P1**。四大基础类型大多可用，但中文离线数据包的重要展示面受损。

## 5. 修复方案

### 方案 A：补齐官方格式解析并建模“不适用”状态

- **做什么**：解析家具、任务、收集包和配方的旧式显示字段、特别订单短占位符和重复分隔符；让料理/制作配方仅在没有显式名时关联其官方产物；将裁缝规则分为独立技术类型，并为日程、商店、掉落等记录标记 `not_applicable`。
- **优点**：直接利用游戏自带中文资源，修复真实数据与报告根因，后续版本可复用。
- **缺点 / 风险**：会触及解析、关联和标准化模块，需要覆盖真实结构回归测试。
- **影响面**：`official.py`、`official_assets.py`、`localization.py`、`game_source.py`、`normalize.py`、报告模型与测试。

### 方案 B：维护静态中文覆盖表

- **做什么**：在人工覆盖文件中录入家具、订单、任务和配方中文。
- **优点**：短期改动少。
- **缺点 / 风险**：重复官方数据，随游戏版本漂移，无法修正报告的技术误报。
- **影响面**：人工数据文件和少量合并逻辑。

### 方案 C：仅过滤报告告警

- **做什么**：把若干类型从缺汉化报告中排除。
- **优点**：告警数量快速下降。
- **缺点 / 风险**：用户可见的家具、订单和任务仍没有中文，掩盖根因。
- **影响面**：仅报告层。

### 推荐方案

**推荐方案 A**，因为它以官方真实字符串为唯一中文来源，同时修复展示数据和报告口径。用户已授权直接实施，视为选择方案 A。
