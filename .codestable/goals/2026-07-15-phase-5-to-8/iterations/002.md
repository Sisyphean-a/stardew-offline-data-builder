# Iteration 002

## Current Understanding

阶段 5 已经把真实 `build` 链路打通到“官方 + 社区 + 报告”。阶段 6 需要把这个输出升级成用户可分发的构建产物：`manifest.json`、数据库哈希和 `.svdata` 包。

## Implementation Approach

- 为数据库写入引入统一 `generated_at`，让数据库、manifest、ZIP 元数据可控。
- 实现 manifest 生成与 `.svdata` 打包。
- 用固定时间测试重复构建一致性，避免把结果建立在系统时钟偶然相同上。

## Changes This Iteration

- 新增 `src/builder/utils/time.py`，统一时间来源。
- 扩展 `write_database`，支持 `generated_at`、`source_hash`、`community_data_version`、`game_version` 元数据。
- 新增 `src/builder/commands/package.py`，负责 `manifest.json` 与 `.svdata` 打包。
- `build` 命令现在会生成：
  - `stardew.db`
  - `manifest.json`
  - `stardew-zh-cn.svdata`
  - `reports/*.json`
- 新增 `tests/test_package.py`，覆盖包结构和固定时间下的重复构建一致性。

## Verification Evidence

- `pytest`：`17 passed`
- `ruff check .`：`All checks passed!`
- 手动 smoke：
  - `python -m builder build --game-dir <fixture-game> --community-data tests\fixtures\community-data --output <dist>`
