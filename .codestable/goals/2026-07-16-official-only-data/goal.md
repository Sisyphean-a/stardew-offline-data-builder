# 官方数据独立构建

> **2026-07-18 发布勘误**：本记录的 `complete` 不再构成数据包发布认证。后续真实资产审计发现旧格式、图片和质量门禁缺陷；见 [当前勘误](../2026-07-18-data-build-pipeline-remediation/completion-claim-correction.md)。

## 目标

彻底移除正式构建对 `stardew-valley-data` 的依赖。游戏事实只来自用户本机的官方解包资产，
通过直接解析和跨表关联形成可查询的 SQLite、图片、报告和 `.svdata`。

## 起点

- 当前不传 `--community-data` 已能构建完整核心产物。
- 生产代码仍包含社区目录参数、加载器、匹配器、合并器和社区版本记录。
- 同版本实测中，社区源不改变 3688 条实体、官方文本、图片或搜索，但会给 1123 条实体补充
  `extra_json` 便利字段。
- `FishPondData.json`、`Machines.json`、`Locations.json` 等官方文件存在，但当前未进入标准化
  enrich 流程。

## 验收标准

1. 正式 CLI、当前文档和运行时代码不再暴露社区数据输入。
2. 删除社区源及其匹配、合并、版本探测、报告字段和测试夹具，不留旧兼容。
3. 可由官方资产获得的事实性便利字段改由官方直接解析或跨表推导，且 provenance 指向官方文件。
4. 官方没有结构化等价物的社区原创摘要和攻略标签不伪造。
5. 真实官方目录独立构建成功，核心实体、文本、搜索、图片和包结构无回归。
6. 自动化检查、真实终端构建和首轮独立代码审查通过；owner 已明确豁免追加复审与独立验收。

## 非目标

- 不反编译游戏程序集来恢复仅存在于代码中的规则。
- 不保留 `--community-data` 兼容入口。
- 不修改历史归档对旧实现的事实记录。
- 不联网获取任何数据。

## 决策与假设

- “摒除依赖”包含公共 CLI、生产代码、当前文档和测试，不包含不可变的 Git 历史与历史验收报告。
- 本地 `aliases.zh-CN.json`、`categories.zh-CN.json` 和空的人工 override 属于项目配置，不是
  `stardew-valley-data` 依赖，可继续保留。
- 选择长期可维护的官方资产解析与关联，不以社区 JSON 复制、硬编码快照或模拟成功替代。
- 当前安装的官方 1.6.15 数据作为真实黑盒验收样本；实现不硬编码该版本的实体计数。

## 当前状态

Goal 已完成。正式构建已移除 `stardew-valley-data` 输入、加载、匹配、合并和版本记录，
事实性便利字段改由官方数据直接解析或跨表推导。

自动化测试、静态检查和真实官方目录构建均通过；首轮独立审查提出的问题已经修复。
owner 于 2026-07-16 明确取消追加代码复审与独立验收，并要求保留当前验证状态收尾。
